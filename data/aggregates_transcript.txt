How do I determine what belongs to an aggregate, i.e., where do the boundaries of this aggregate lie?

If I assume that I use R to call my Root Entity and want to check for another Entity i, whether it is an inner Entity.

Then I can formulate four rules that help me determine whether R and i are in the same aggregate.

The first rule is that i, i.e., the inner Entity, always occurs together with R, i.e., the Root Entity.

That means there is no query for i except together with a query for R, i.e., the Root.

The second rule is that all accesses from the outside to i take place via R.

So it doesn't even make sense to access i if R is not included in the context.

The third condition is that if R is deleted, i is always deleted.

And the fourth condition, as an optional condition, is that there may be invariants between R and i and possibly other classes.

So conditions that are formulated between these Entities and that always apply.
